# World Cup Live Pulse 本轮总任务清单

更新：2026-07-12

这是本轮唯一执行清单。产品目标是做成可被球迷长期使用、可被评委完整体验、具备商业延展能力的参赛作品，而不是堆叠功能的展示网页。

## 本轮重新锁定的任务目标

球迷价值顺序固定为：

1. 比分、比赛状态和数据校验时间足够快、足够可信。
2. 进球、红黄牌、比分复核、半场和完场等关键事件不漏且容易查看。
3. 今日赛程、已完成比赛和晋级路径一眼可懂。
4. 错过比赛后可用时间线、关键节点、AI 摘要和官方回放入口快速补课。
5. 比分挑战、关注提醒、语音解说和分享历史形成重复使用动力。

本轮交付门槛：

- [x] 完成三轮独立市场调研：实时比赛中心、赛程 / 晋级 / 回放、AI / 个性化 / 互动留存。
- [x] 每轮调研都必须形成“市场证据 -> 球迷痛点 -> 产品修改或明确不做”的记录。
- [x] 至少完成三轮功能验收，并且三轮都针对同一个候选发布版本。
- [x] 第一轮验证数据真实性与刷新；第二轮验证完整球迷流程与多语言；第三轮验证生产构建、安全和移动端；线上同 SHA 复验仍是独立发布门槛。
- [x] 最终 Demo 录制 A/B 两版已完成，每版少于五分钟；A 为 3:00，B 为 3:20。
- [x] 两版 Demo 均完成两遍全片音视频解码、旁白裁切检查和首中尾抽帧复核；B 版作为最终上传候选。
- [x] 不为阵容、xG、伤停、新闻、赔率或直播视频建立无来源占位。

## 0. 不可妥协的产品原则

- [x] 主用户是普通球迷，首屏只展示比赛、比分、状态、挑战、关键事件和 AI 解读。
- [x] 官方要求只是最低线；每项能力必须扩展成 2 至 3 个可演示、可测试、对球迷有价值的闭环。
- [x] 没有可靠来源的字段不显示，不猜测、不补零、不显示“待更新 / 待补充 / 即将上线”等假深度占位。
- [x] 当前比赛、未来赛程和历史回放严格分区，旧数据只能进入明确标注的回放空间。
- [x] 比分挑战只使用本地积分，无现金价值，不连接钱包，不下注，不提供交易或投注建议。
- [x] 只提供 FIFA / FIFA+ 等官方观看、集锦、赛况入口；不抓取或嵌入非授权直播。
- [x] Token、JWT、`.env.local`、私钥、助记词和验证码永不进入仓库、公开前端、截图或演示视频。

## 1. 核心球迷主流程

- [x] 比赛中心首屏突出双方、比分、比赛状态、数据校验时间和一句话 AI 赛况。
- [x] 1,000 本地初始积分的比分挑战占据主内容最高交互权重。
- [x] 预测按比赛 ID 保存，一场只扣一次积分，只有可靠完场结果才能结算一次。
- [x] 展示等级、经验、连胜、已产生结果后的命中率和可核验历史记录。
- [x] 提供分享 / 下载预测卡，但不把积分描述为资产、奖金或筹码。
- [x] 用户刷新页面后，积分、预测、结算与历史记录保持一致。
- [x] 赛季演示与真实本地积分账本隔离，不能伪装成用户真实参与历史。
- [x] 最终在线版本再次验证一次“选择比分 -> 锁定 -> 完场结算 -> 刷新不重复奖励”完整闭环。

## 2. 实时数据与真实性

- [x] TxLINE 仅接收世界杯 `CompetitionId 72`，明确拒绝 Friendlies `CompetitionId 430`。
- [x] 当前选中比赛每 15 秒刷新；页面重新获得焦点或恢复可见时立即刷新。
- [x] 未结算挑战在后台定时复核，使用 `game_finalised` 作为最终结算依据。
- [x] 进球、黄牌、红牌、比分复核、半场、完场和时间节点均从规范化事件生成。
- [x] 推翻的临时进球作为比分复核处理，不计入确认进球。
- [x] 官方赔率为空时整个数值区域隐藏，不生成模拟赔率。
- [x] 数据请求禁用缓存冻结，来源状态和校验时间对球迷可读。
- [x] 保留 8 场无凭据的 2026 TxLINE 历史回放，保证非比赛时段仍可完整评审。
- [x] 发布前连续执行两次真实 TxLINE 探测，并把当次结果记录到工作日志，不把波动值写成永久事实。

## 3. 赛程、晋级与回放

- [x] 赛程页区分当前 / 即将开始与已完成回放。
- [x] 当前赛程直接来自活动适配器；没有可靠当前赛程时整段隐藏。
- [x] 回放卡展示真实双方、阶段、日期、最终比分和可用事件，不展示空字段。
- [x] 提供 32 强、16 强、8 强、4 强、决赛、冠军等赛事路径视图，只呈现已被来源确认的结果。
- [x] 每场回放支持时间线、关键节点跳转、播放速度、进度和 AI 解说。
- [x] 提供 FIFA+ 官方回放库、集锦和官方世界杯赛况入口。
- [x] 加入“无剧透回放”入口，让用户可从第 1 分钟开始而不先看到最终比分。
- [x] 最终核对所有阶段、日期、比分、晋级关系与来源，任何未确认关系必须隐藏。

## 4. 球队与球员详情

- [x] 球队页只使用 2026 已认证回放推导比赛数、胜场、进失球、黄红牌和直接回放入口。
- [x] 球员只展示来源提供的姓名或 ID 及其已确认参与事件，不自封“关键球员”。
- [x] 移除旧 2022 主动球队图鉴、虚构教练、占位阵容和待补充信息。
- [x] 比赛中心保留与当前比赛直接相关的紧凑球员事件；完整记录放在球队详情。
- [ ] 有可靠阵容、伤停、位置、球衣号或统计端点后才启用对应字段；当前没有来源则不建设空壳 UI。
- [x] 最终语言复核球队名、阶段名、状态和球员记录，避免半中半英。

## 5. AI 球迷解说与互动

- [x] AI 解说只基于当前规范化比分和事件，不引用页面外事实。
- [x] 支持进球、黄牌、红牌、比分复核、半场、完场和动量变化的多语言摘要。
- [x] 支持浏览器语音播报，并在不可用时保持文字功能完整。
- [x] 提供关注 / 取消关注比赛按钮；关注后仅对新验证事件提示。
- [x] 浏览器通知不可用或被拒绝时，站内关注状态仍然可用。
- [x] 为关注按钮、通知状态、语音控制和动态事件补齐自动化交互测试。
- [x] 最终演示证明 AI 文字、语音、事件来源和比分不会互相矛盾。

## 6. 多语言、无障碍与视觉质量
