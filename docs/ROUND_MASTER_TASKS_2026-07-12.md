## A. 完成定义

- [x] 普通球迷打开首屏即可看懂比赛、状态、比分挑战、关键事件和 AI 解读。
- [x] 每项官方评审要求至少延展为 2 至 3 个真实可用、可测试的球迷功能。
- [x] 所有可见事实有可靠来源；无来源字段整段隐藏，不猜测、不补零、不用占位制造丰富度。
- [x] 当前比赛、未来赛程、2026 已完成比赛和历史演示严格分区。
- [x] 同一工作树通过数据、可用性、语言/移动端/安全三轮本地验收。
- [x] 本地验收后制作有感情的英文 Demo，并完成两轮自动化与逐场景音画复检。
- [x] 最终视频与提交材料事实一致后发布线上版本，并完成在线回归检查。

## B. 球迷主流程

- [x] 比赛中心突出双方、比分、状态、核验时间和一句话 AI 赛况。
- [x] 1,000 本地积分比分挑战是最高权重交互；不连接钱包、无现金价值、不是下注。
- [x] 预测按比赛保存，只扣一次积分，只由可靠完场结果结算一次。
- [x] 实时模式只在开赛前接受预测；锁定后仍可在开赛前免费修改，开赛后自动关闭，历史仍保持每场一条记录。
- [x] 等级、经验、连胜、命中率和历史记录可刷新持久化。
- [x] 回放支持时间线、关键节点、速度、无剧透和官方 FIFA/FIFA+ 入口。
- [x] 本地收藏球队会优先赛程、球队和下一场入口，不隐藏其他比赛。
- [x] 复核每一个主按钮真实可用，删除无动作、重复或只用于解释产品的按钮。

## C. 数据真实性与自动更新

- [x] TxLINE 只接收世界杯 `CompetitionId 72`，拒绝 Friendlies `CompetitionId 430`。
- [x] 当前比赛每 15 秒刷新，页面恢复焦点或可见时立即刷新。
- [x] 进球、红黄牌、比分复核、半场、完场、加时节点均由规范化事件生成。
- [x] 推翻的临时进球不计入确认进球；只有 `game_finalised` 可结算挑战。
- [x] 官方赔率为空时隐藏整个赔率区域，不生成模拟赔率。
- [x] 2026 TxLINE 已验证回放用于非比赛时段评审，并与实时状态明确区分。
- [x] 在最终候选上连续运行两次 TxLINE 探测，记录当时结果但不把动态值写成永久事实。
- [x] 逐场核验可见球队、阶段、比分、事件、日期、晋级和来源时间。

## D. 赛程、晋级、球队与球员

- [x] 赛程页区分当前/即将开始与 2026 已完成回放。
- [x] 晋级路径只呈现来源已确认的结果；未确认轮次留白或隐藏。
- [x] 球队记录由已验证比赛推导比赛数、胜场、进失球和红黄牌。
- [x] 任何球员内部 ID 不得作为姓名展示；来源只有 ID 时保留事件与球队，不建立球员档案。

- [x] 全站清除 `#数字`、`Player 123`、`Goal event`、`Card event` 等球迷不可读表达。
- [x] 阵容、伤停、位置、球衣号、xG、评分等没有可靠端点时不显示。

## E. AI 解说与自然语音

- [x] AI 内容只依据当前规范化比分与事件，不引用页面外事实。
- [x] 提供“现场快报 / 为什么重要 / 30 秒补课”三个互补模式。
- [x] 未开赛时三个模式分别提供对阵/阶段/开球、晋级意义/挑战规则和赛前补课，不再复用一句“未开赛”。
- [x] 关注比赛后，只对新验证事件触发站内或浏览器提示。
- [x] 已验证回放优先播放本地授权声线预生成音频，缺失时降级为优化后的浏览器语音。
- [x] 网页解说文案与屏幕比分、事件和语言一致；中英文六段均通过反向识别、时长、响度、静音和削波检查。
- [x] 最终 Demo 声音按场景有适度情绪：开场和挑战有能量，关键事件有临场感，补课有共情，数据与安全沉稳，结尾自信。
