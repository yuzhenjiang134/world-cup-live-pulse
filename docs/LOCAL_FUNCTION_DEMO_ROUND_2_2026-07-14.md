# 本地完整功能展示：第二轮

日期：2026-07-14  
视角：连续球迷任务与恢复能力  
候选：当前本地生产构建

## 连续任务

本轮不按组件逐个点击，而是模拟一名球迷从第一次进入到赛后回看的完整使用过程。

1. **赛前**：识别下一场比赛和开球时间，提交积分比分挑战，关注比赛并选择进球、红黄牌与完场提醒。
2. **比赛中**：在比分和 Pulse Play 之间切换；查看当前节点、场上人数和最新已确认事件；在对应球队看台留下本地评论。
3. **错过片段**：使用“30 秒补课”和关键事件按钮快速回到上下文，不需要阅读开发数据。
4. **赛后**：确认完场状态，挑战只结算一次；进入赛程与回放查看已验证 2026 归档，使用无剧透时间线复盘。
5. **继续探索**：查看来源支持的球队记录和已确认晋级路径；没有可靠资料的部分留白或隐藏。
6. **跨设备尺寸**：在桌面和 390px 移动端重复主路径，确认导航、22 人剧场、看台和挑战均可操作。
7. **刷新恢复**：刷新页面后，挑战记录、收藏、提醒偏好、看台评论和反应仍存在于本设备。

## 可恢复性与安全边界

- 实时访问不可用时，界面明确降级为已验证回放或赛前状态，不把旧数据标成直播。
- 官方赔率缺失时整块隐藏，不生成猜测值。
- 视频只提供合法的官方入口或项目方配置的授权地址，不抓取非授权直播。
- 评论和积分均为本地体验，不登录、不接钱包、无现金价值、不产生交易建议。
- 浏览器语音不可用时不会阻断比分、事件、挑战和文字补课主流程。

## 第二轮结论

本地完整球迷旅程通过。核心任务没有依赖开发者诊断页，也没有需要解释才可使用的占位按钮；剩余发布门槛是同一提交的线上复检和项目所有者对最终 Demo 录制的明确批准。
